An Offline-First Swedish Accounting Engine Built For Zero Trust

Designed to conform strictly with Sweden's rigorous Bokföringslag (BFL) and GDPR. Engineered as an atomic desktop application using Go and SQLite, secured with database-level WORM integrity and memory-safe offline micro-engines.

Clean Local Architecture

LocalLedger shifts away from standard cloud subscription accounting. It runs with a fully isolated kognitive-offline architecture where data ownership is returned entirely to the user.

  • No Cloud Vectors: Prevents vendor-lockin and blocks network-based database breaches.
  • Zero-Dependency Onboarding: Single-executable deployment with stateless config checks.
  • BFL-Compliant: Immutable data storage using physical database seals and triggers.
  • WORM File Storage: System backups are cryptographically secured using AES-256 ZIP layers.

		       EXISTS(SELECT 1 FROM verifications WHERE storno_ref_id = v.id) as is_stornoed,

// decodeSIE4 automatically sniffs and decodes the encoding of the input stream.
// Since a standard CP437 encoded file containing Swedish characters (å, ä, ö)
// is mathematically invalid under the UTF-8 standard, we leverage utf8.Valid 
// to distinguish the stream with absolute mathematical precision.
func decodeSIE4(fileData []byte) ([]byte, string) {
	hasNonASCII := false
	for _, b := range fileData {
		if b > 127 {
			hasNonASCII = true
			break
		}
	}

	if hasNonASCII && utf8.Valid(fileData) {
		return fileData, "UTF-8"
	}

	// Stream is non-UTF8 with active characters -> Decodes strictly as IBM PC8 (CP437)
	decoder := charmap.CodePage437.NewDecoder()
	decoded, err := decoder.Bytes(fileData)
	if err != nil {
		return fileData, "UTF-8 (Fallback)"
	}
	return decoded, "CP437"
}
Key Engineering Highlight: Automatic encoding sniffing removes standard user-friction. Using utf8.Valid over binary buffers permits flawless parsing of Swedish national letters from classic legacy accounting software exports.

The Adversarial Hardening Ledger

A curated log of critical system bugs, design adjustments, and hard security fixes logged iteratively during the development lifecycle.

WORM_COMPLIANCE_01 Security

WORM (Write Once Read Many) Ledger Sealing

Enforced transactional immutability directly in the database engine using native SQLite BEFORE UPDATE and BEFORE DELETE triggers.

Security Hardening: Ensures full Swedish Bokföringslag compliance. Even if a bad actor manages to inject rogue Go application calls, the underlying SQL storage engine strictly rejects any modification of historical audit tracks.
INV-012 Architecture

TOCTOU Race Condition Prevention

Replaced deferred transaction loops with explicit table write-locks. Handled credits via a portable db-mutex, locking sum evaluations before commit.

Security Hardening: Under multi-threaded conditions, a user could double-submit credit invocations to bypass legal refund caps. Table-locking enforces strict atomicity.
POST_AUDIT_HARDEST_FIX_01 Security / XSS

Stored XSS Vector Hardening in Logo Upload

Integrated a magic-byte MIME detector and an offline SVG-XSS scanning parser blocking embedded JavaScript execution in company logos.

Security Hardening: SVG files are valid XML and can carry malicious scripts. Uploaded SVGs are dynamically inspected for active payload nodes, combined with strict CSP nosniff and sandbox headers during serving.
EULA_HARDENING_01 Security / UX

Clickwrap EULA Escape Prevention

Bound keyboard events and modal panels with x-bind:inert focus locks, blocking global Command Palette (Ctrl+K) bypasses.

Security Hardening: Ensures absolute legal compliance before any local API bootstrapping begins. Postpones complete runtime initialization until the client explicitly consents.
PORTABLE_LAUNCHER_02 Robustness

Cascading Config and Path Traversal Shielding

Introduced a fallback config sequence that switches write profiles to temp directories if running off read-only USB media, backed by strict workspace input path cleaners.

Security Hardening: Prevents unauthorized path traversal attacks while facilitating flawless offline portable setups across varying Windows disk letters.
